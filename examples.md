# Neural Conjecture Generation: Best Examples Explained

This document presents the most impressive conjectures generated by our neural
models, translated from TPTP/Mizar notation into human-readable mathematics.
Each example includes the problem context, the generated lemma, and why it helps.

---

## How to Read These Examples

The prover E works on problems in Clausal Normal Form (CNF). Each clause is a
disjunction (OR) of literals. The negated conjecture to prove appears as clauses
marked `negated_conjecture`. The prover tries to derive a contradiction.

- **L** = number of clauses E processes to find a proof (baseline, no help)
- **L1** = clauses processed when the conjecture is added as an extra axiom
- **L2** = clauses processed to prove the conjecture itself from the problem
- **Speedup** = L / (L1 + L2) --- how many times faster the split proof is

## 1. Function Composition is Subset-Preserving (t13_funct_2)

**Speedup: 106x** (L=60,393 down to 567) | Method: C (VAE) | Split: train

**The Problem**: Given functions f: A->B and g: B->C, both total and functional
(well-defined), prove a property about their composition. The problem involves
the domain and range relationships of composed functions with subset conditions.

**Generated Conjecture**:
```
The image of f restricted to any subset is itself a subset of the
range-domain product of that image.
```

In symbols: for any set X, the relational image f[X] is a member of the powerset
of the Cartesian product of its own range. This is essentially saying "the image
of a function is well-typed" --- a basic fact about function images that the prover
needs as an intermediate step but takes 60,000 clauses to discover on its own.

**Why it helps**: The prover gets stuck exploring the space of function composition
properties. This lemma provides the key typing fact that lets it short-circuit
the search. The function image being well-typed is obvious to a mathematician but
hard for the prover's heuristic search to prioritize.

## 2. Closure of Interior is Contained in Closure (t26_tops_1)

**Speedup: 81x** (L=14,330 down to 177) | Method: A3 (150ep) | Split: train

**The Problem**: In a topological space, prove that applying closure-interior-closure-interior
to a set gives the same result as closure-interior. Formally:
Cl(Int(Cl(Int(A)))) = Cl(Int(A)) for any subset A of a topological space.

**Generated Conjecture**:
```
For any subset A of a topological space X:
  Cl(Int(A)) is contained in Cl(A)
```

This says: "the closure of the interior of A is a subset of the closure of A."
Written as: Cl(Int(X,A)) ⊆ Cl(X,A), provided A is a subset of the carrier of X
and X is a topological space.

**Why it helps**: This is a fundamental topological monotonicity fact --- the interior
shrinks a set, so its closure is no bigger than the closure of the original. The
prover needs this as a stepping stone but doesn't discover it efficiently. Four
different methods independently generated variants of this lemma, confirming it's
the natural key insight.

**Remarkable**: The model discovered this topological principle from graph structure
alone, without knowing what "closure" or "interior" mean. It recognized the
structural pattern: an operator applied twice can be simplified using a monotonicity
property of the inner operator.

## 3. Partial Sums Form a Function (t42_series_3)

**Speedup: 67x** (L=32,791 down to 488) | Method: C (VAE) | Split: train

**The Problem**: Prove a convergence property of a series of real-valued functions.
The series involves partial sums and absolute values.

**Generated Conjecture**:
```
The partial sums of a series form a function (are well-defined),
provided the absolute value of the series is an extended real number.
```

In Mizar terms: k1_seq_1(k3_series_1(s), n) is a function (v1_funct_1) whenever
|k3_series_1(s)| is an extended real and n is a natural number.

**Why it helps**: The prover spends most of its 32,000 clauses establishing basic
well-definedness properties of the partial sum sequence. This lemma directly
provides that fact, letting the prover focus on the actual convergence argument.

## 4. Evaluation of Partial Functions (t13_partfun2)

**Speedup: 65x** (L=14,566 down to 223) | Method: A+named | Split: train

**The Problem**: Prove a property about partial function evaluation --- when you
evaluate a partial function at a point, the result belongs to the function's range.

**Generated Conjecture**:
```
The evaluation of function f (restricted via partfun2) at point c
is a member of the relational image of the domain of f.
```

This states that applying a partial function to an argument in its domain yields
a value in the function's codomain. A basic "function typing" fact.

**Why it helps**: Similar to Example 1 --- the prover needs this typing lemma as
scaffolding but can't find it efficiently. The model learned that function evaluation
problems need typing lemmas about the range.

## 5. Set Difference Decomposition (t35_xboole_1) --- TEST SET

**Speedup: 16.5x** (L=39,681 down to 2,410) | Method: A3 (150ep) | Split: **test**

**The Problem**: Given A ⊆ B and C ⊆ D, prove that A \ D ⊆ B \ C.
(If A is contained in B and C is contained in D, then the difference
A minus D is contained in B minus C.)

**Generated Conjecture**:
```
For any element x and any sets A, B:
  x ∈ A  OR  x ∈ B  OR  x ∉ A \ B
```

This is the set difference membership lemma: "if x is in A \ B, then x is in A."
Equivalently: x ∈ A, or x ∈ B, or x ∉ (A \ B). This is a tautology of set theory.

**Why it helps**: The prover needs to decompose set difference expressions in the
goal. This lemma provides the fundamental decomposition rule: membership in A \ B
implies membership in A and non-membership in B. Without it, the prover must
reconstruct this from the definition of set difference, which requires many steps.

**This is a test-set result** --- the model never saw this problem during training,
yet generated exactly the right lemma. The same pattern (set difference decomposition)
appeared in the training data for other problems, and the model transferred it.

## 6. Ordinal Transitivity (t13_ordinal1) --- TEST SET

**Speedup: 26.5x** (L=15,084 down to 568) | Method: A3 (150ep) | Split: **test**

**The Problem**: If element a belongs to ordinal b, then a is also an ordinal.
(Ordinals are transitive: membership in an ordinal implies being an ordinal.)

**Generated Conjecture**:
```
Either b is a subset of any set X, or some element belongs to a.
```

Formally: r1_tarski(esk2_0, X1) | r2_hidden(X2, esk1_0), where esk2_0 is the
ordinal b and esk1_0 is the element a.

**Why it helps**: This seems oddly general, but it provides a key case split that
the prover needs. Either the ordinal b is trivially small (subset of everything),
or there exist elements witnessing the structure of a. This case analysis is the
crux of the ordinal transitivity proof.

## 7. Constant Function Image (t87_funcop_1)

**Speedup: 58x** (L=31,269 down to 543) | Method: A3 (150ep) | Split: train

**The Problem**: Prove that constant functions on disjoint domains are compatible
(agree on their overlap, which is empty). Involves the funcop_1 theory of
function operations.

**Generated Conjecture**:
```
For any element x and sets A, B:
  x ∈ range(A) ∩ B  OR  x ∉ constfunc(A, B)
```

This says: if x is in the constant function mapping A to B, then x must be
in the intersection of A's range and B. It's a basic fact about how constant
functions behave with respect to Cartesian products.

**Why it helps**: The prover needs to reason about what elements look like inside
constant functions. This lemma provides the key structural decomposition.

## 8. MML Query Composition (t49_mmlquery) --- TEST SET, D-only

**Speedup: 21x** (L=10,417 down to 494) | Method: D (SSM) | Split: **test**

**The Problem**: Prove a property about composition of mathematical database queries
in the MML (Mizar Mathematical Library) query system. Two queries composed in
different orders should satisfy a relational property.

**Generated Conjecture**:
```
The composition of query Q1 after filter F applied to Q2
satisfies the relational property with the composition of
filter F applied to Q1 after filter F applied to Q2,
unless Q1's filter is trivial.
```

**Why it helps**: This is a commutativity-like property for MML queries that the
prover needs as a lemma. Only the D (SSM) model found this --- no other method
generated a useful conjecture for this problem.

**Remarkable**: This is a highly domain-specific lemma about a library query system,
yet the model discovered it from structure alone. The SSM's "adventurous" generation
style explores unusual structural patterns that other models don't.

## 9. Simplex Membership (t6_simplex0) --- TEST SET

**Speedup: 12.7x** (L=3,509 down to 277) | Method: A2 (100ep) | Split: **test**

**The Problem**: Prove that if a set family A is a subfamily of B
(every member of A is contained in some member of B), then the simplex
(abstract simplicial complex) of A is contained in the simplex of B.

**Generated Conjecture**:
```
For any element x:
  x ∈ simplex(B)  OR  x ∉ A
```

This says: any element of the original family A must appear in the simplex of B.
It's the key monotonicity property connecting set families to their simplicial
complexes.

**Why it helps**: Simplex constructions involve complex recursive definitions.
This lemma cuts through by providing the direct membership implication.

## 10. Relational Image of Constant Functions (t31_partfun2)

**Speedup: 52x** (L=12,717 down to 243) | Method: A2 (100ep) | Split: train

**The Problem**: Prove a property about how constant functions interact with
relational images and function evaluation.

**Generated Conjecture**:
```
The image of a constant function constfunc(A, x) applied to set B
equals the image of constfunc(A, y) applied to B,
provided A is non-empty and the function f is relational.
```

Paraphrase: constant functions with different values produce the same relational
image structure (up to the constant value). This is because the relational image
depends on the domain, not the value.

**Why it helps**: This equivalence lemma lets the prover simplify expressions
involving constant functions, avoiding a combinatorial explosion in the search.

## 11. Complement of the Universe (t12_tops_3)

**Speedup: 39x** (L=64,708 down to 1,675) | Method: D+named | Split: train

**The Problem**: Prove a property about continuous functions between topological
spaces, involving preimages of open/closed sets.

**Generated Conjecture**:
```
The complement of the carrier set within itself equals the carrier set,
OR the carrier set is a member of the powerset of its own complement.
```

This is essentially saying that the complement of the universe in itself is a
trivial set --- a boundary case that the prover needs to handle but takes 64,000
clauses to work out from definitions.

**Why it helps**: Topological proofs often need to handle the boundary case where
a set is the entire space. This lemma provides that case directly.

## 12. Real Number Trichotomy (t17_xreal_1) --- TEST SET

**Speedup: 38x** (L=8,380 down to 219) | Method: D+named | Split: **test**

**The Problem**: Prove an inequality property about extended real numbers.

**Generated Conjecture**:
```
Either X1 is not an extended real, or X2 is not an extended real.
```

This looks almost trivially simple --- it says at least one of two things isn't
a real number. But in the context of proof by contradiction (where the prover
assumes everything IS a real number), this creates a crucial case split.

**Why it helps**: The prover's negated conjecture assumes both variables are real.
This conjecture forces the case analysis that the prover needs to structure its
search. Sometimes the most useful lemma is the simplest one.

## 13. Euclidean Complex Norm (t40_euclid_3) --- TEST SET

**Speedup: 41x** (L=3,760 down to 92) | Method: C (VAE) | Split: **test**

**The Problem**: Prove a property about complex numbers embedded in Euclidean
space --- relating the real and imaginary parts.

**Generated Conjecture**:
```
The real part of the first number is a real number,
OR the imaginary part of the second number is not a real number.
```

**Why it helps**: The problem involves relating complex number components to real
number properties. This lemma provides the typing fact that real/imaginary parts
are themselves real numbers --- again, a fact obvious to mathematicians but hard
for the prover to prioritize.

## 14. Ordinal Arithmetic Cancellation (t7_arytm_3)

**Speedup: 36x** (L=16,089 down to 442) | Method: A3 (150ep) | Split: train

**The Problem**: Prove a property about ordinal arithmetic --- specifically about
subtraction and ordering of ordinals.

**Generated Conjecture**:
```
(a - (a - b)) / a = a  (in ordinal arithmetic)
OR  a - b = 0
OR  b is not less than a
OR  b is not a natural number
```

This is an ordinal arithmetic cancellation law with appropriate side conditions.

**Why it helps**: Ordinal arithmetic is notoriously difficult for automated provers
because the operations don't have the same algebraic properties as regular arithmetic.
This lemma provides a cancellation rule that shortcuts the algebraic reasoning.

## 15. File Record Sequence Properties (t25_filerec1)

**Speedup: 39x** (L=10,323 down to 262) | Method: D (SSM) | Split: train

**The Problem**: Prove properties about file records modeled as finite sequences
over natural numbers.

**Generated Conjecture**:
```
The natural numbers are non-empty, OR the file record is empty,
OR the natural numbers are non-empty.
```

This is a degenerate-case lemma: the natural numbers are non-empty (always true),
which provides a trivial but essential fact. The repetition of the condition
(natural numbers non-empty appears twice) makes this a tautology that the prover
can use immediately.

**Why it helps**: Sometimes the prover needs a trivially true fact stated in the
right form to trigger its simplification rules. The model learned to generate
these "structural tautologies."

## Patterns Observed

### 1. Typing Lemmas Are King
The most common pattern among high-speedup conjectures is **typing lemmas**: facts
stating that certain expressions are well-defined, are members of the right type,
or belong to the right domain. Examples: #1, #3, #4, #13. These are obvious to
mathematicians but hard for provers because the type information is implicit in
the problem structure.

### 2. Case-Split Catalysts
Several conjectures work by providing case analyses that help the prover structure
its search: #6, #12. These may look trivial but they create the right branching
structure for the proof search.

### 3. Monotonicity and Containment
Topological and set-theoretic proofs often need monotonicity lemmas (#2, #5, #9):
if you apply an operator to a smaller set, you get a smaller result. The model
learned this general pattern and applies it across theories.

### 4. Algebraic Simplification
For algebraic problems, the model generates cancellation laws and simplification
rules (#14). These shortcut the prover's equational reasoning.

### 5. Cross-Theory Transfer
The model generates structurally similar lemmas for structurally similar problems
across different mathematical domains. The set difference decomposition (#5),
topological monotonicity (#2), and function typing (#1, #4) patterns each appear
in multiple contexts, confirming that the anonymous GNN encoder successfully
captures transferable mathematical structure.

### 6. The SSM Finds Unique Solutions
The D (SSM) model finds speedups that no other model discovers (examples #8, #15).
Its sequential state-space model explores different parts of the generation space
than the Transformer, making it valuable for ensemble approaches.
